git常用命令整理

git

git常用命令

查看git版本

git –version

生成公钥

ssh-keygen -t rsa -C “wang-qz@foxmail.com“

检查密匙是否成功绑定

ssh -T git@github.com
ssh -T git@gitee.com

测试ssh连接是否成功

ssh -v git@github.com

git配置用户和邮箱

git config –global user.name “wang-qz”
git config –global user.email “wang-qz@qq.com“

查看git配置的用户和邮箱

git config user.name
git config user.email

将 color.ui 设置为 auto 可以让命令的输出拥有更高的可读性

git config –global color.ui auto

查看git配置信息

git config –list

把本地目录变成Git可以管理的仓库

git init

查看当前分支的状态(git bash打开的目录必须是git管理的仓库目录)

git status

添加远程仓库的关联,url为你的远程仓库的url，可采用http协议或ssh协议

远程仓库的名称一般默认为origin,也可以自定义, 下面同时关联gitee和github时就自定义区分了不同平台的远程仓库名

git remote add
git remote add origin git@github.com:wang-qz/help-doc.git
git remote add origin git@gitee.com:wang-qz/leangit.git
git remote add origin https://gitee.com/wang-qz/leangit.git

查看关联的远程仓库的名称

git remote

查看关联的远程仓库的详细信息

git remote -v

第一次需要将本地分支关联到远程分支

先将远程分支拉取下来

git pull origin master

然后关联本地分支和远程分支

git branch –set-upstream-to=origin/master master
git pull -–rebase origin master
git push origin master

修改远程仓库的关联

git remote set-url
git remote set-url origin https://gitee.com/wang-qz/leangit.git
git remote set-url origin git@gitee.com:wang-qz/leangit.git

删除远程仓库的关联

git remote remove
git remote remove origin
git remote rm origin

本地仓库同时关联gitee和GitHub的远程仓库及分支, 分别远程仓库名称自定义, 不能同时命名为origin

git remote add gitee git@gitee.com:wang-qz/leangit.git
git remote add github git@github.com:wang-qz/learngit.git
git pull gitee master
git pull github master
git branch –set-upstream-to=gitee/master master
git branch –set-upstream-to=github/master master
git pull -–rebase gitee master
git pull -–rebase github master

推送到远程仓库

可以把 master 换成你想要推送的任何分支

-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令

git push -u origin 对应远程分支名
git push -u origin master

如果已经关联了远程分支,可以不加-u参数

git push origin master

同步远程最新内容到本地

git pull

协同办公,远程仓库已有多次提交记录,可以使用rebase模式更新

git pull –rebase

如果rebase异常,取消rebase

git reflog
git reset –hard HEAD@{1}
git rebase –abort

从远程仓库克隆工程 ssh或http

git clone git@github.com:wang-qz/help-doc.git
git clone https://github.com/YOUR-USERNAME/Spoon-Knife

添加文件到暂存区 stash

git add fileName
git add filename1 filename2 filename3
git add .

commit文件到本地仓库 后面是提交的注释说明

git commit -m “first commit”

dev分支工作没做完不想提交, 但需要在新的分支上修复bug, 可以将dev分支没做完的工作暂存, 等修复完bug再回到dev分支恢复暂存到工作区继续工作

git stash

查看暂存的未提交内容

git stash list

恢复未提交内容到工作区 apply-恢复不删除暂存 drop-恢复并删除暂存

git stash apply
git stash drop

将修复的bug(b5d5245)快速合并到dev分支

git cherry-pick b5d5245

查看工作区和本地仓库最新版本的区别 HEAD是本地仓库最新版本

git diff HEAD – hello.txt

查看本地仓库提交的历史记录日志

git log

每一条提交记录只占一行的输出

git log –pretty=oneline

只看某一个人的提交记录

git log –author=wang-qz

树形结构来展示所有的分支提交日志记录

git log –graph –oneline –decorate –all
git log –graph –pretty=oneline –abbrev-commit
git log –graph –oneline –abbrev-commit

查看提交记录

git log –name-status

打开日志帮助文档

git log –help

在本地创建和远程分支对应的分支

git checkout -b branch-name origin/branch-name

创建+切换分支

git checkout -b feature_x
git switch -c feature_x

上面相当于下面两句

git branch feature_x
git checkout feature_x 或 git switch feature_x

查看本地分支

git branch

查找本地分支

git branch | grep branchName
git branch | findstr branchName

查看远程分支

git branch -r
git branch –remote

查找远程分支 -r 或 –remote

git branch -r | grep branchName
git branch -r | findstr branchName

查看本地+远程分支

git branch -a
git branch -a | grep branchName
git branch -a | findstr branchName

切换回主分支

git checkout master
git switch master

再把新建的feature_x分支删掉

git branch -d feature_x
git branch –delete feature_x

强制删除

git branch -D feature_x

删除本地分支和远程分支关联

git branch -r -D origin/feature_x

删除远程分支

git push origin –delete feature_x

切换文件 可以丢弃工作区的修改

git checkout – filename
git restore – filename

把readme.txt文件在工作区的修改全部撤销

git checkout – readme.txt
git restore – readme.txt

合并其他分支feature_x到你的当前分支master

先切换到master分支,再将其他分支合并进来

git checkout master
git merge feature_x

禁用fast forward模式合并

git merge –no-ff -m “merge message” feature_x

如果merge过程中发生冲突报错, 可以取消本次merge,后面重新合并

git merge –abort

在合并改动之前，你可以使用如下命令预览差异

git diff
git diff master feature_x

工作区的当前版本从本地仓库恢复到上一个版本 HEAD^表示上一个版本

HEAD^^表示上上一个版本，HEAD~100上100个版本

git reset –hard HEAD^

从本地仓库恢复到工作区, 可以直接用commitId指定

git reset –hard commitID
git reset –hard 088bb37

可以把暂存区的修改撤销掉，重新放回工作区

从暂存区恢复到工作区(没有commit到本地仓库)

git reset – readme.text
git reset – HEAD readme.txt
git reset HEAD readme.txt
git restore –staged readme.txt

撤销工作区的修改

git restore – readme.txt
git checkout – readme.txt

直接将文件删除且添加到暂存区(staged)

git rm filename

上面删除操作可以用下面两步代替

rm filename
git add filename

文件内容中文乱码问题

git默认utf-8编码,所以如果要上传的文件不是中文乱码,需要使用utf-8编码编辑文件内容

git bash 终端中文乱码

git config –global core.quotepath false
